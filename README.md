## 📖 À propos

La **FISA** (Formation d'Ingénieur par l'apprentissage en Système Alternance) Informatique est un cursus d'ingénieur proposé par le **CESI**, école d'ingénieurs généraliste.

| Détail | Valeur |
|--------|--------|
| 🏫 École | CESI |
| 📚 Filière | FISA Informatique |
| 🔄 Rythme | Alternance (entreprise / école) |
| 📅 Promotion | 2024 – 2027 |
| 🎯 Niveau | Bac+5 – Titre d'ingénieur |

Cette organisation regroupe les projets pédagogiques et personnels réalisés tout au long du cycle.

## 🤝 Comment contribuer

Tous les membres de la promo sont invités à contribuer aux dépôts communs. Merci de respecter les conventions suivantes.

### 📌 Nommage des dépôts

| Type | Convention | Exemple |
|------|-----------|---------|
| Projet de cours | `<numéro>-<NomDuCours>` | `01-Big-Data` |
| Projet de groupe | `<sujet>-groupe-<n>` | `iot-groupe-2` |
| Projet libre | `<PrénomNom>_<Projet>` | `Loic-Sacha_BattleShip` |

### 🌿 Branches

```
main        → branche stable, protégée
dev         → branche de développement principale
feat/<nom>  → nouvelle fonctionnalité
fix/<nom>   → correction de bug
docs/<nom>  → mise à jour de documentation
```

### 📬 Pull Requests

1. Créer une branche depuis `dev` (ou `main` si `dev` n'existe pas).
2. Nommer la branche selon la convention ci-dessus.
3. Ouvrir une PR avec un titre clair et une description des changements.
4. Demander la revue d'au moins un autre membre de la promo.
5. Merger uniquement après approbation.

### ✅ Bonnes pratiques

- Rédiger un `README.md` dans chaque dépôt.
- Ajouter une licence (de préférence MIT) pour les projets publics.
- Commiter régulièrement avec des messages explicites (`feat: ajout de la page d'accueil`).
- Ne jamais commiter de secrets, tokens ou données sensibles.
